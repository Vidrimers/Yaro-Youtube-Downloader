# Plan: Cookie Rotation System

## Goal
Support up to 10 cookie sets per platform (Instagram + YouTube). Bot automatically rotates to the next working set when current cookies expire. Admin can upload/manage all 10 slots via Telegram.

## File Naming Convention
- Instagram: `instagram_cookies_1.txt` ... `instagram_cookies_10.txt`
- YouTube: `cookies_youtube_1.txt` ... `cookies_youtube_10.txt`

## Changes

### 1. `config/config.js` — Replace single paths with arrays
- Replace `INSTAGRAM_COOKIES_FILE` with `INSTAGRAM_COOKIES` (array of 10 paths)
- Replace `YOUTUBE_COOKIES_FILE` with `YOUTUBE_COOKIES` (array of 10 paths)
- Keep backward compat: if old env var set, put it in slot 1
- Load from env: `INSTAGRAM_COOKIES_1` ... `INSTAGRAM_COOKIES_10`, same for YouTube
- Default paths if not in env: `/home/ytdownload/instagram_cookies_N.txt` and `/home/ytdownload/cookies_youtube_N.txt`

### 2. `bot.js` — New state and cookie manager
- Add `this.instagramCurrentIndex = 0` and `this.youtubeCurrentIndex = 0`
- Add `this.instagramCookiesAlertSent = false` per-slot tracking (Map or Set)
- Add helper method `getNextWorkingCookies(platform)`:
  - Start from current index, try each slot up to 10
  - Test with `yt-dlp -j --cookies <path> <testUrl>` (short timeout)
  - Return first working path, update current index
  - If none work → notify admin, return null

### 3. `bot.js` — Health checks → try all slots
- `startInstagramCookiesHealthCheck()`: iterate all 10 slots, test each
  - If current slot fails → auto-rotate to next working one
  - If all fail → single admin notification "Все 10 куков Instagram протухли"
  - If some work → notify admin which slots expired, which still work
- `startCookiesHealthCheck()` (YouTube): same logic

### 4. `bot.js` — Download functions use current working cookies
- `processInstagramDownload`: use `this.config.INSTAGRAM_COOKIES[this.instagramCurrentIndex]`
- `processVideoDownload`: use `this.config.YOUTUBE_COOKIES[this.youtubeCurrentIndex]`
- If download fails with `INSTAGRAM_LOGIN_REQUIRED` → auto-rotate to next slot and retry once
- Same pattern for YouTube 403 errors

### 5. `bot.js` — Admin menu restructure
- Button "🍪 Куки Instagram" → callback `admin_cookies`
  - Shows 10 inline buttons: `🍪 Instagram #1` ... `🍪 Instagram #10`
  - Each shows current status (working/expired/empty) via indicator
- Button "🍪 Куки YouTube" → callback `admin_youtube_cookies`
  - Shows 10 inline buttons: `🍪 YouTube #1` ... `🍪 YouTube #10`

### 6. `bot.js` — Upload flow per slot
- New callback pattern: `admin_cookies_insta_1` ... `admin_cookies_insta_10`
- `admin_cookies_yt_1` ... `admin_cookies_yt_10`
- `handleCookiesUpload` accepts slot number, saves to correct path
- After upload → test the slot → report result

### 7. `bot.js` — Admin notifications
- On health check failure: "Instagram куки #3 и #7 протухли. Работают: #1, #5"
- Include buttons to update specific expired slots
- On auto-rotation: "Instagram: переключился с #3 на #5"

### 8. `src/api.js` — Update extension API
- Replace `this.config.INSTAGRAM_COOKIES_FILE` → `this.config.INSTAGRAM_COOKIES[currentIndex]`
- Replace `this.config.YOUTUBE_COOKIES_FILE` → `this.config.YOUTUBE_COOKIES[currentIndex]`

### 9. Migration
- Rename existing files on server:
  - `instagram_cookies.txt` → `instagram_cookies_1.txt`
  - `cookies.txt` → `cookies_youtube_1.txt`
- Deploy script or manual SSH commands
